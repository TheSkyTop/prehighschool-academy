# PreHighschool Academy: Two-Week Commercial Pilot Design

## Goal

Build and test 14 daily learning missions that feel close to commercial release quality before expanding to the remaining 50 weeks.

Each day must include:

- One focused topic.
- Structured learning content for Year 4-6 students.
- One visual model or diagram.
- One embedded or approved YouTube video candidate.
- Ten quiz questions tied directly to the learning content.
- One exam strategy and one common-trap check.

## Year-One Scale

- 7 subjects per week.
- 52 topics per subject.
- 364 yearly topics.
- 10 quiz questions per topic.
- 3,640 yearly quiz questions.

The first two weeks should become the content quality template for the full year.

## Daily Lesson Design

Each daily lesson should follow the same structure:

1. Big idea: one sentence that tells the child what matters.
2. Core method: 4-6 concise steps or concepts.
3. Worked demo: show the method in action.
4. Graphic demo: bar, chart, table, diagram, map, passage annotation, or puzzle grid.
5. Exam strategy: how this appears in selective, scholarship, 11+, or extension tests.
6. Common traps: 2-4 mistakes children commonly make.
7. Quick check: one simple confidence check before quiz.
8. Video support: one approved primary video plus one backup search/video.

## Ten-Question Quiz Blueprint

Every daily quiz should use this shape:

| Question | Purpose | Style |
|---|---|---|
| Q1 | Recall key concept | quick multiple choice |
| Q2 | Recall language/formula | fill, cloze, or choice |
| Q3 | Basic application | direct question |
| Q4 | Visual/diagram question | chart, graph, passage, table, or image |
| Q5 | Core exam application | selective-style |
| Q6 | Multi-step reasoning | scholarship-style |
| Q7 | Common trap | identify error |
| Q8 | Data/evidence question | justify from source |
| Q9 | Timed challenge | harder but fair |
| Q10 | Boss question | extension or writing response |

Writing days should still have 10 tasks, but Q10 should be a 100-200 word response. Earlier questions should build the plan, vocabulary, sentence control, and evidence.

## Subject Quality Rules

### Maths

Use mathematical notation inside worked calculations, not as isolated theory. Every topic should include at least one visual model such as:

- Number line.
- Percentage bar.
- Pie chart.
- Area model.
- Coordinate grid.
- Data chart.

Quiz should include calculation fluency, word problems, trap questions, and one selective-style multi-step problem.

### Reading

Reading topics should use short, child-accessible passages inspired by:

- Classic literature.
- History.
- Civics and political ideas.
- Economics and society.
- Science non-fiction.

Quiz should ask for evidence, inference, vocabulary in context, author's purpose, tone, and best-supported answer.

### English

English should cover grammar, sentence control, punctuation, vocabulary, cloze, and verbal reasoning.

Quiz should include:

- Cloze choices.
- Sentence repair.
- Word form.
- Synonym/antonym.
- Prefix/root/suffix.
- Exam-style verbal reasoning.

### Writing

Each writing day should prepare children for Australian, UK, and US exam-style responses.

Use:

- Strong model paragraph.
- Planning frame.
- Sentence upgrade.
- Evidence/example builder.
- 100-200 word final response.

Optional image or video prompts can increase motivation, but sharing must remain parent-approved only.

### Science

Science should split clearly into:

- Biology.
- Chemistry/materials.
- Earth and space.
- Scientific inquiry and data.

Every lesson should include a diagram, table, experiment setup, or evidence model.

### Physics

Physics should use clear visual models:

- Force arrows.
- Motion graphs.
- Circuit diagrams.
- Light rays.
- Sound waves.
- Energy transfer chains.

Quiz should include misconception checks, diagram interpretation, and applied problems.

### Reasoning

Reasoning should include:

- Verbal reasoning.
- Non-verbal reasoning.
- Logic.
- Computational thinking.
- Data and finance.
- Basic trading/risk reasoning.

Finance content must be educational only: probability, risk, charts, and decision-making. It should not give investment advice.

## YouTube Recommendation Rules

Each topic should store three video candidates:

1. Primary: best fit, free, clear, age-appropriate, embeddable.
2. Backup: same topic if primary fails.
3. Stretch: deeper video for advanced students.

Ranking formula:

`score = topic match 40% + age fit 20% + exam usefulness 15% + clarity 10% + duration 10% + embeddable/free 5%`

Preferred free channels:

- Maths: Math Antics, Math Mammoth, Khan Academy, BBC Bitesize, Numberock.
- Science/Physics: Crash Course Kids, SciShow Kids, National Geographic Kids, TED-Ed, BBC Bitesize.
- English/Reading: BBC Bitesize, Khan Academy Grammar, Nessy, Storyline Online for reading interest.
- Reasoning/Computing: BBC Bitesize Computing, TED-Ed logic puzzles, Code.org-style concepts.

## Storage And Generation Strategy

Store as little as possible, but never at the cost of quality or child safety.

### Store permanently

- Curriculum graph: subject, strand, topic, prerequisite, difficulty.
- Lesson template schema.
- Quiz template schema.
- Rubric and marking rules.
- Approved video metadata.
- Generated weekly pack version hash.
- Parent/child progress and mistake history.

### Generate or cache weekly

- Daily learning text.
- Ten quiz questions.
- Explanations and hints.
- Diagrams as structured data or SVG/HTML models.
- Video ranking from approved candidate lists.

### Do not rely on live scraping in the child app

Live web scraping each week is risky because:

- Content can disappear or change.
- Copyright and licensing are uncertain.
- Video embed availability changes.
- Web pages may contain unsafe or irrelevant material.
- Latency and network cost are unpredictable.

Recommended model:

1. Server job generates next week's pack from the knowledge map.
2. Server checks source/video candidates.
3. Pack is cached and versioned.
4. Human or automated QA validates age fit, answer correctness, and safety.
5. App downloads only the approved compact weekly pack.

```ts

  week: number;

};

type WeeklyPack = {
  id: string;
  week: number;
  version: string;
  generatedAt: string;
  topics: GeneratedTopic[];

```

The app should load one approved `WeeklyPack` at a time, plus progress history. This keeps storage small and makes updates easy.

## Tomorrow's Acceptance Test

The two-week pilot passes only if:

- All 14 topics have complete learning content.
- Every topic has exactly 10 quiz questions.
- Every quiz question has answer, explanation, difficulty, exam style, and mistake tag.
- At least one visual appears per topic.
- Video candidates are free and relevant.
- Rewards correctly give 100 points per completed day.
- The content feels useful for Australian selective/scholarship preparation while also mapping to UK and US standards.
